This page is kept for compatibility with older runbooks and bookmarks.

For first-time cloud onboarding, this tutorial is obsolete. Use <a href="/docs/tutorials/launch-from-released-cloud-image">Launch Neuwerk From The Released Cloud Image</a> as the primary path.

## When This Compatibility Page Is Still Useful

Use this page only if you are:

- translating an older "single node" procedure into the current cloud-image flow
- validating a legacy environment that still references this title
- comparing old command expectations against current service defaults

If you are starting fresh, go directly to the cloud-image tutorial and come back here only for historical mapping.

## Legacy-to-Current Mapping

Older single-node guidance usually mapped to these outcomes:

- boot one appliance instance
- confirm management and metrics endpoints
- mint an initial admin token
- create first policy

Those outcomes now live in two canonical docs:

- instance bootstrap and verification: <a href="/docs/tutorials/launch-from-released-cloud-image">Launch Neuwerk From The Released Cloud Image</a>
- first enforcement workflow: [Create Your First Policy](/docs/tutorials/create-your-first-policy)

## If You Must Validate An Existing Single-Node Runtime

When checking an environment that was built from older instructions, verify:

- management API health endpoint responds
- metrics endpoint responds
- management and data interfaces are distinct
- DNS target/upstream settings are present
- token mint flow works for operator bootstrap

For concrete endpoint and bootstrap command details, rely on the cloud-image tutorial so your validation steps stay aligned with the supported distribution path.
