This page is retained for compatibility with older docs links and internal runbooks.

For first-time cloud onboarding, this tutorial is obsolete. Start with [Launch Neuwerk From The Released Cloud Image](/docs/tutorials/launch-from-released-cloud-image) and treat this page as migration context only.

## Why This Page Still Exists

Older guides often used "build a two-node cluster" as the first deployment step. The supported flow has changed:

- first boot and baseline validation should follow the cloud-image tutorial
- rollout behavior for autoscaling groups should follow [Cloud Rollout Integration](/docs/architecture/cloud-rollout-integration)
- cluster topology and failure boundaries should follow [Run An HA Cluster](/docs/deployment/high-availability)

## Legacy Cluster Checks You Can Reuse

If you inherited an existing two-node environment, keep these verification checks:

- each node reports healthy and ready state
- leader election is stable
- replicated policy state converges on both nodes
- state-changing API calls succeed when routed through either management endpoint

These checks remain valuable even though this page is no longer the recommended starting point.

## Migration Note For Existing Runbooks

If your runbook currently says:

- "deploy single node"
- "build two-node cluster"
- "then create first policy"

Update it to:

1. launch from released cloud image
2. [create first policy](/docs/tutorials/create-your-first-policy)
3. apply cloud rollout integration guidance
4. scale and validate HA behavior

## Next Steps

- Start from [Launch Neuwerk From The Released Cloud Image](/docs/tutorials/launch-from-released-cloud-image) if onboarding a new deployment.
- Read [Cloud Rollout Integration](/docs/architecture/cloud-rollout-integration) before ASG/VMSS/MIG-based scaling.
- Use [Run An HA Cluster](/docs/deployment/high-availability) and [Cluster Replication](/docs/architecture/cluster-replication) for cluster operation details.
