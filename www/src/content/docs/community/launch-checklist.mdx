Use this page as the final OSS launch preflight for the public project surface.

## Repository Surface

Confirm the repository root exposes the expected public files:

- Apache 2.0 `LICENSE`
- `SECURITY.md`
- `CONTRIBUTING.md`
- the canonical release-readiness summary

## Appliance Release Surface

Confirm the appliance release path is ready:

- `.github/workflows/image-release.yml` is present
- the shared public signing key is tracked
- signed `SHA256SUMS` are part of the release contract
- appliance release docs cover verification and import flow

Relevant pages:

- [Release Process](/docs/community/release-process)
- [OSS Release Readiness](/docs/community/release-readiness)

## Terraform Provider Surface

Confirm the Terraform provider release path is ready:

- provider source address remains `moolen/neuwerk`
- public signed releases come from `moolen/terraform-provider-neuwerk`
- the monorepo still owns packaging and release-source contract checks

## Manual Launch Blockers

These items are still manual and must be handled outside this repo preflight:

- configure appliance release signing secrets in GitHub Actions
- complete Terraform Registry onboarding for `moolen/neuwerk`
- run the first signed appliance image release
